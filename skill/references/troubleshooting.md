# 异常处理指南

## API 解析失败

| 现象 | 原因 | 处理 |
|------|------|------|
| HTTP 500 | 抖音 WAF 拦截 / 服务端异常 | 重试1次，仍失败则告知用户「解析服务暂不可用，稍后再试」 |
| HTTP 403 | 身份码无效/已停用 | 检查身份码是否正确传入 |
| HTTP 429 | 每日/总次数用完 | 引导购买套餐续费 |
| 超时（>30s无响应） | 抖音图文笔记解析慢 | 重试1次，设置 60s 超时 |

## 素材下载失败

| 现象 | 原因 | 处理 |
|------|------|------|
| 视频URL下载返回403 | CDN 防盗链/URL过期 | 直接用 `data.url` |
| 动图段下载失败 | 抖音 CDN geo-block / 过期 | 跳过这段动图，只合剩下的段 + 静图 |
| 图片下载失败 | 代理临时故障 | 重试1次，跳过该张图片 |
| 配乐下载失败 | CDN 问题 | 跳过配乐，合无声音轨版本 |

## 动图合成失败

| 现象 | 原因 | 处理 |
|------|------|------|
| 脚本输出 `匹配: {}` | URL格式变更，匹配链全挂 | 检查 `get_img_filename` 函数，看 base64 解码是否成功 |
| ffmpeg 报错 / 未安装 | 环境缺 ffmpeg | 先 `apt install -y ffmpeg` 安装，仍不行则**退到发静图** |
| 合成视频时长异常 | 静图段缺音轨导致 concat 截断 | 检查脚本是否正确处理了 `anullsrc` |
| 输出文件不存在 | 脚本中途崩了 | 检查 stderr 输出，复现问题 |
| 文件 > 20MB | 动图段码率高 | 当前 CRF 18 视觉无损，如文件超限可临时调高 CRF（23~28），或减少 CYCLES（3→2） |

## 🆘 兜底方案：合成失败 → 发静图

如果 `synthesize_live.py` 因任何原因合成失败（ffmpeg 未安装、脚本报错、素材下载不全）：

**直接下载 `data.images[]` 所有图片，逐张发 `MEDIA:` 给用户。** 虽然没动图，但至少能看到内容。

```bash
# 逐张下载并发送
for i in "${!images[@]}"; do
  curl -so /tmp/img_${i}.jpg "${images[$i]}"
  # 然后发 MEDIA:/tmp/img_${i}.jpg
done
```

> 静图兜底优于什么都不发。用户有内容看，AI 也不至于卡住。
