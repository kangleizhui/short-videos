# 身份码使用规则

## 身份码提示（按 `identity_info` 显示）

| identity_info | 显示 |
|----------|------|
| `daily_remaining: 8` | `📊 今日免费剩余：8次（每身份码每天10次）` |
| `remaining: 42` | `📊 剩余：42次` |
| `remaining: null, expires_at: 有值` | 不显示剩余（时长/永久密钥） |
| `daily_remaining: 0, remaining: null` | `📊 今日免费次数已用完。可以购买套餐继续使用（发送"套餐"查看）` |

## 📌 身份码规则

- 身份码是5位字母数字，**大小写不敏感**
- **永久不变**，购买套餐后同一码叠加权益
- 可以随时修改成更好记的码（见 modify_identity.php）
- 同 IP + 同设备指纹只会返回同一个身份码，防刷

## AI 行为规则

1. **永远只有一个「当前使用的身份码」** — 首次调用自动生成后保存
2. **开始对话** → 不传 code 调 `/api/parse.php?url=链接` → 自动生成身份码 → 从响应中提取并保存
3. **用户买了套餐** → 查支付结果 → 权益自动绑定到当前身份码 → 告知用户
4. **调用失败（403 / 429）** → 按 HTTP 码区分处理：

| HTTP | 含义 | 操作 |
|------|------|------|
| 403 | 身份码无效/已停用 | 检查身份码是否正确传入 |
| 429 | 每日/总次数用完 | 引导购买套餐续费 |
| 500 | 解析服务异常 | 重试一次，仍失败告知用户 |

## 📊 权益优先级

**包月/包天/永久(不限量) → 免费每日(10次/天) → 按量付费(计次)**

| 用户有什么 | 实际消耗顺序 |
|-----------|------------|
| 只有免费每日 | → 直接用免费每日 |
| 免费 + 按量 | → 先用免费10次/天 → 用完了扣按量 |
| 包月 + 免费 + 按量 | → 先用包月不限量 → 包月到期 → 回退免费10次/天 → 免费用完 → 扣按量 |
| 已过期套餐 | → 冻结次数自动转成按量，同上逻辑 |

如果用户问"到期了会怎样"，答案永远是：先回退免费每日，免费用完了才扣按量，不会直接废掉。

## 🔍 last_ip 说明

`last_ip` 记录的是**谁调了 API**，不是最终用户的IP。因为 Agent/机器人是从本机调 API，所以通常显示为 `127.0.0.1`。真实用户直接调 API 时才会记录用户公网 IP。

## 常见翻车

| 错误 | 纠正 |
|------|------|
| 不保存身份码直接调下一次 | **必须从首次响应的 `identity_info` 中提取身份码并保存** |
| 手动新建多个身份码 | **一个就够了**，所有权益叠加到同一个码 |
| 传小写身份码以为不行 | ✅ 大小写都支持，服务端自动转大写 |
| 猜权益优先级搞反了 | ⚠️ **不限量 → 免费每日 → 按量计次**，忘了回来查 |
