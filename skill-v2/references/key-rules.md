# 身份码规则

## 什么是身份码

- 5位大写字母数字（避开 0/O/1/I/L 等易混淆字符）
- 示例：`X7K3M`、`G8NJ2`、`W5T9A`
- 首次调用 API 自动生成（不传 key 参数即可）
- 同一用户永久使用同一个身份码

## 为什么用5位身份码

| 旧版（长密钥） | 新版（5位身份码） |
|----------------|-------------------|
| `TEST1-ABCD-EFGH-IJKL-MNOP` | `X7K3M` |
| 35位长度 | 5位长度 |
| AI 容易截断加 `...` | 绝对不会截断 |
| 固定测试密钥不安全 | 每个用户独立身份码 |
| 格式复杂易输错 | 5位大写，轻松记忆 |

## 权益叠加规则

一个身份码可以拥有多个权益，自动优先生效：

| 权益类型 | 说明 | 优先级 |
|----------|------|--------|
| `free_daily` | 每天10次免费体验 | 最低 |
| `count` | 购买的计次次数 | 中 |
| `monthly` / `daily` | 包月/包天无限次 | 高 |
| `lifetime` | 永久无限次 | 最高 |
| `lifetime_daily` | 永久每日限额 | 高 |

权益自动优先级：`lifetime` > `monthly`/`daily`/`lifetime_daily` > `count` > `free_daily`

## 购买流程

1. 用户有身份码（免费阶段获得）
2. 用户选择套餐 → AI 下单 `POST /pay/create.php`
3. 生成二维码 → 用户扫码支付
4. 支付成功后，服务端自动将权益叠加到身份码
5. **同一身份码不变**，权益自动升级

## 常见错误

- ❌ 以为身份码是"密钥"需要手动输入 → 首次用自动生成
- ❌ 生成新身份码给同一用户 → 不行，必须用回原来的
- ❌ 手动截断身份码（如 `X7K...`）→ 5位这么短截断什么？
